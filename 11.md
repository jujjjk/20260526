一、项目背景
==================================================

我现在有一个 12-DoF 四足机器人，结构类似 A1 / Go2，每条腿 3 个关节：

- hip
- thigh
- calf / knee

当前训练框架：

- IsaacLab
- RSL-RL
- 机器人配置继承或参考 A1 / Go2 速度跟踪任务
- 当前部署在真机上，Jetson / ROS2 / ONNX / 电机控制链路已经基本跑通

之前纯 RL 策略在真机上出现以下问题：

1. 最新 ONNX 后，部署链路基本正常，不是旧 ONNX 问题；
2. cmd=0 时动作基本正常；
3. cmd_x=0.12 时，policy raw action 明显饱和；
4. FR 腿三关节长期 raw action 超过 ±1；
5. 每帧平均约 3 个以上关节被 clip；
6. FR 腿 action 饱和率明显高于 FL / RL / RR；
7. q_error 不算大，说明电机基本能跟上目标角；
8. 但下地仍触发保护，说明目标轨迹本身不合理；
9. 当前问题主要在训练策略，而不是 ROS / 电机通信 / ONNX 导出链路。

已有数据结论大致如下：

cmd_x = 0.12 时：

- FR_hip raw>|1| 约 79%
- FR_thigh raw>|1| 约 69%
- FR_calf raw>|1| 约 65%
- FR leg raw>|1| 约 71%
- FL leg raw>|1| 约 30%
- RL leg raw>|1| 约 9%
- RR leg raw>|1| 约 0%
- clip_count mean 约 3.3 joints/frame

说明：

- 策略不是动作太小，而是动作分布不合理；
- 策略依赖 clip / safety 兜底；
- 四腿输出不平衡，FR 特别异常；
- 下地触发保护很可能来自不合理接触轨迹、前腿硬顶、持续高力矩或局部堵转。

因此本次目标不是继续纯 RL 硬调，而是引入：

    CPG 基础步态 + RL residual 小修正 + 电机真实约束 + 柔顺输出链路

核心动作形式：

    q_target = q_cpg + delta_q_rl

其中：

- q_cpg 由 CPG 生成基础小跑节律；
- delta_q_rl 是 RL 输出的小幅 residual；
- residual 必须严格限幅；
- 最终输出必须经过 joint limit / rate limit / low-pass / delay / torque clamp；
- 训练环境和真机部署必须尽量使用同一套动作后处理逻辑，避免 sim-to-real 不一致。

==================================================
十一、第八部分：奖励函数补充
==================================================

请在 mdp_rewards.py 或当前 reward 文件中新增以下 reward/penalty。

全部必须可通过 cfg 开关和权重控制。

1. action saturation penalty

目的：
让 raw action 自然落在可执行区间，不依赖 clip。

公式：

    sat = relu(abs(raw_action) - 0.75)
    penalty = sum(sat^2)

不要等到超过 1 才罚，从 0.75 左右就开始软惩罚。

1. residual magnitude penalty

CPG residual 模式下，惩罚 delta_q_rl 过大，避免 RL 覆盖 CPG。

    penalty = sum(delta_q_rl^2)

3. residual rate penalty

惩罚 residual 帧间变化：

    penalty = sum((delta_q_rl_t - delta_q_rl_t_minus_1)^2)

4. action acceleration penalty

惩罚 action 二阶差分：

    penalty = sum((a_t - 2*a_t_minus_1 + a_t_minus_2)^2)

5. torque penalty

惩罚整体力矩：

    penalty = sum(torques^2)

6. torque rate penalty

惩罚力矩突变：

    penalty = sum((torques_t - torques_t_minus_1)^2)

7. torque limit penalty

接近安全力矩阈值时惩罚：

    penalty = sum(relu(abs(torque) - soft_torque_limit)^2)

8. per-leg action balance penalty

统计四条腿 residual RMS：

    FR = action[:, 0:3]
    FL = action[:, 3:6]
    RL = action[:, 6:9]
    RR = action[:, 9:12]

    leg_rms = mean(each_leg_action^2)
    penalty = var(leg_rms)

注意：权重不要太大，避免压死正常步态差异。
目标是避免 FR 长期远大于其他腿。

1. foot clearance reward

摆动相足端达到 step_height 附近给奖励。
不要鼓励过度抬腿。

1. foot slip penalty

支撑相足端接触时，惩罚足端 xy 速度。

1. gait phase reward

trot 模式下：

- 鼓励 FR + RL 接触节律相近；
- 鼓励 FL + RR 接触节律相近；
- 鼓励两组相差半周期。

这项先给小权重，避免强行锁死策略。

1. joint limit margin penalty

惩罚关节长期贴近限位。

1. base orientation penalty

保持 roll / pitch 稳定，避免身体倾斜后 policy 输出激进。

1. feet air time / contact regularity

鼓励每条腿都有合理摆动和支撑，不允许某条腿长期偷懒或长期硬顶。

目标：
让 policy 学会自然柔顺，而不是靠 clip、filter、safety 强行变柔顺。

    先让机器人学会一个电机能承受的、四腿均衡的、低速柔顺小跑，
    而不是让纯 RL 继续靠 FR 腿打满输出和 clip 去硬撑。
