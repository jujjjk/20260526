六、第三件事：训练端新增 diagonal support switch reward
============================================================

请在 mdp_rewards.py 新增一个奖励/惩罚函数，例如：

phase_diagonal_support_switch_penalty

目标：

当 CPG 相位要求 FR+RL 支撑时：
  FR+RL 的支撑代理应该大于 FL+RR

当 CPG 相位要求 FL+RR 支撑时：
  FL+RR 的支撑代理应该大于 FR+RL

注意：
1. 不要要求差距过大，否则会让动作僵硬。
2. 不要强行要求永远两脚接触，因为动态 trot 允许短暂过渡。
3. 用 smooth hinge loss：
   penalty = relu(margin - (expected_diag_proxy - other_diag_proxy))
4. margin 先小一点，例如 0.05~0.10，根据 proxy 归一化尺度调整。
5. 该 reward 要有 cfg 开关和权重。
6. 默认权重保守，例如 -0.05 到 -0.15，不要一开始太大。

如果 IsaacLab 训练里有 contact sensor，优先用 contact force。
如果没有可靠 contact sensor，可以先用 estimated torque 或 foot contact proxy。

同时新增 diagnostic-only reward：

diagonal_support_accuracy_metric

返回 0/1 或连续值，用于 TensorBoard 观察：

- FR+RL support accuracy
- FL+RR support accuracy
- average diagonal support accuracy

特别要单独记录：

FL_RR_support_success_rate

因为当前问题主要是 FL+RR 接力不足。

============================================================
七、第四件事：新增 phase-aware hip residual gate
============================================================

当前不是 hip 没有动作，而是 residual 可能破坏 CPG 支撑语义。

请在训练端 cpg_actions.py 中新增函数：

apply_phase_aware_hip_residual_gate(...)

部署端也必须实现同样逻辑，参数一致。

核心原则：

CPG 决定哪组腿支撑；
RL residual 只能在这个语义附近微调；
不允许 residual 把支撑腿推向明显内收；
不允许 residual 让摆动腿继续强外展。

定义：

hip indices:
FR_hip = 0
FL_hip = 3
RR_hip = 6
RL_hip = 9

side_sign:
FR = -1
FL = +1
RR = -1
RL = +1

outward amount:
outward = side_sign * (q_hip - q_default_hip)

其中：
- 对右腿 FR/RR，q 变负通常是外展，所以 side_sign=-1
- 对左腿 FL/RL，q 变正通常是外展，所以 side_sign=+1
如果真机语义测试发现方向相反，只改 cfg 里的 hip_balance_signs，不要改函数逻辑。

规则：

1. 支撑相腿：
   outward_final 不允许低于 stance_min_outward。
   建议默认：
   stance_min_outward = 0.005 或 0.010 rad

2. 摆动相腿：
   outward_final 不允许高于 swing_max_outward。
   建议默认：
   swing_max_outward = 0.030 或 0.040 rad

3. residual 不允许反向盖掉 CPG 的支撑语义。
   如果 q_raw = q_cpg + delta_q_rl 后导致支撑腿 outward_final < stance_min_outward，则修正 delta_q_rl。

4. 摆动腿不要继续强外展。
   如果 swing 腿 outward_final > swing_max_outward，则修正 delta_q_rl。

5. 所有修正都要平滑，不要硬跳。
   可以用 clamp，但要记录 clamp 数量。

6. 新增诊断：
   - hip_gate_clamp_count
   - hip_gate_clamp_ratio
   - hip_stance_inward_violation
   - hip_swing_over_outward_violation
   - FR/FL/RR/RL hip outward before gate
   - FR/FL/RR/RL hip outward after gate

配置项建议放在 CPGCfg 或 action cfg：

enable_phase_aware_hip_gate = True
hip_gate_stance_min_outward = 0.008
hip_gate_swing_max_outward = 0.035
hip_gate_side_signs = (-1.0, 1.0, -1.0, 1.0)
hip_gate_debug = True

非常重要：
训练端和部署端必须共用同一套默认参数。
不能训练时没有 gate，部署时才加 gate。

请在 cpg_cfg.py / rough_env_cfg.py 中改为：

或者先用 0.04。

不要同时放大 hip_amp 和 residual_limit_hip。
当前优先级是让 CPG 主导相位，residual 只小修。
