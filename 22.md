==================================================
补充：随机性必须结合 URDF 自动生成推荐范围
==================================================

请不要手写固定随机范围，而是新增一个工具：

    tools/generate_randomization_from_urdf.py

输入：

1. fanfan URDF；
2. 当前 robot cfg / default_joint_pos；
3. motor_profile.yaml；
4. motor_profile_fitted.yaml，如果存在；
5. CPGCfg，如果存在。

输出：

    config/randomization_profile.yaml
    logs/randomization_from_urdf_report.txt
    logs/randomization_from_urdf_report.yaml

目标：
根据 URDF 的真实几何、质量、惯量、关节限位和腿长，自动生成适合本机器人的 domain randomization 初值。所有范围都必须可在 cfg 中手动覆盖。

一、从 URDF 提取的内容

请解析并保存：

1. links:
   - link name
   - mass
   - inertia ixx/iyy/izz/ixy/ixz/iyz
   - inertial origin xyz/rpy
   - visual/collision geometry 尺寸，如果可读

2. joints:
   - joint name
   - parent link
   - child link
   - joint axis
   - joint origin xyz/rpy
   - lower / upper limit
   - effort limit
   - velocity limit
   - dynamics damping / friction，如果存在

3. body geometry:
   - base/trunk 尺寸估计
   - base mass
   - base inertia
   - base CoM
   - nominal body height

4. leg geometry:
   - 每条腿 hip/thigh/calf joint
   - thigh length
   - calf length
   - nominal leg length
   - nominal foot position
   - foot link / foot collision size

5. joint safety:
   - joint range = upper - lower
   - default_joint_pos 到上下限距离
   - joint limit margin
   - 哪些关节 default pose 过于接近限位

如果 URDF 里 mass/inertia/limit 缺失或明显异常，必须 WARNING，不要假装可靠。

二、基于 URDF 设置模型随机性

1. mass randomization

根据 URDF link mass 设置随机范围：

- base/trunk mass:
    初始 ±5% ~ ±10%
    如果 URDF 质量不确定，先 ±5%
    不要一开始 ±20% 以上

- leg link mass:
    初始 ±3% ~ ±8%
    calf/foot 更保守

输出示例：

    mass_randomization:
      base_mass_scale: [0.92, 1.08]
      thigh_mass_scale: [0.95, 1.05]
      calf_mass_scale: [0.95, 1.05]
      foot_mass_scale: [0.95, 1.05]

2. inertia randomization

根据 URDF inertia 对角项缩放：

- 初始 ±5% ~ ±10%
- inertia 必须保持正定；
- 不要直接乱改非对角项；
- 如果 inertia 数值异常，输出 WARNING，并禁用 inertia randomization。

输出示例：

    inertia_randomization:
      base_inertia_scale: [0.92, 1.08]
      leg_inertia_scale: [0.95, 1.05]

3. center of mass randomization

根据 trunk 尺寸或 nominal leg length 设置 CoM 偏移：

- base CoM x/y:
    取 trunk length/width 的 2%~5%
    或不超过 0.005~0.015 m

- base CoM z:
    取 body height 的 1%~3%
    或不超过 0.005~0.010 m

不要用固定大数值。小型机器人 CoM 偏移过大会导致训练不稳定。

输出示例：

    com_randomization:
      base_com_x_m: [-0.01, 0.01]
      base_com_y_m: [-0.008, 0.008]
      base_com_z_m: [-0.005, 0.005]

4. joint zero offset randomization

根据每个关节 range 生成零位误差：

    zero_offset = min(0.03 rad, 0.03 * joint_range)

初始推荐：

- hip: ±0.02 ~ ±0.03 rad
- thigh: ±0.02 ~ ±0.04 rad
- calf: ±0.02 ~ ±0.04 rad

如果某关节 default pose 距离限位很近，要自动收紧。

输出示例：

    joint_zero_offset_randomization:
      FR_hip: [-0.025, 0.025]
      FR_thigh: [-0.030, 0.030]
      FR_calf: [-0.030, 0.030]

5. reset joint position noise

根据 joint range 和 default_joint_pos 到限位的距离设置：

    reset_noise = min(0.05 rad, 0.05 * joint_range, 0.5 * limit_margin)

不要让 reset 后 joint 超限。

输出示例：

    reset_joint_position_noise:
      hip_rad: [-0.03, 0.03]
      thigh_rad: [-0.04, 0.04]
      calf_rad: [-0.04, 0.04]

6. joint damping / friction randomization

如果 URDF 中存在 dynamics damping/friction：

    damping_scale: [0.7, 1.3]
    friction_scale: [0.7, 1.3]

如果 URDF 缺失 dynamics：

- 不要假装知道真实摩擦；
- 给一个保守 additive 初值；
- 并标记 WARNING。

输出示例：

    joint_dynamics_randomization:
      damping_scale: [0.8, 1.2]
      friction_scale: [0.8, 1.2]
      additive_friction_nm:
        hip: [0.0, 0.03]
        thigh: [0.0, 0.05]
        calf: [0.0, 0.05]

7. motor strength randomization

结合 motor_profile_fitted.yaml：

如果存在真实拟合数据，优先用 fitted 值。
如果没有，使用保守初始：

    motor_strength_scale: [0.65, 1.0]

但需要根据 URDF effort limit 和 RS01 safe torque 检查：

    effective_torque_limit = min(URDF effort limit, motor safe torque)

如果 URDF effort 比电机说明书更大，使用电机 safe torque；
如果 URDF effort 比说明书更小，输出 WARNING，询问是否 URDF effort 设置过低。

输出示例：

    motor_strength_randomization:
      global_scale: [0.65, 1.0]
      per_leg_scale: [0.75, 1.0]
      per_joint_scale:
        hip: [0.75, 1.0]
        thigh: [0.65, 1.0]
        calf: [0.65, 1.0]

8. torque limit randomization

根据 motor_profile 的 safe_training_torque 设置：

    torque_limit_episode_scale: [0.7, 1.0]

并且绝对上限不能超过 safe_training_torque，不使用 peak_torque 作为长期上限。

输出示例：

    torque_limit_randomization:
      scale: [0.7, 1.0]
      absolute_train_limit_nm: 5.0

9. Kp/Kd randomization

结合当前 cfg 的 Kp/Kd 设置：

初始：

    kp_scale: [0.85, 1.15]
    kd_scale: [0.85, 1.15]

如果真机数据表明 q_error 较大或容易保护，收紧到：

    [0.9, 1.1]

输出示例：

    pd_gain_randomization:
      kp_scale: [0.85, 1.15]
      kd_scale: [0.85, 1.15]

10. action delay randomization

URDF 不能推导延迟。
必须从 motor_profile_fitted 或真机日志估计。

如果没有 fitted 数据：

    action_delay_frames: [0, 3]

如果控制频率已知，例如 50 Hz / 100 Hz，需要换算成 ms 并写入报告。

输出示例：

    action_delay_randomization:
      frames: [0, 3]
      estimated_ms: [0, 30]

三、基于 URDF 设置环境随机性

1. terrain height noise

根据 step_height 和 nominal leg length 设置：

    terrain_noise <= 0.25 * step_height
    同时 <= 0.02 * nominal_leg_length

第一版建议更保守：

    0.0 ~ 0.01 m

输出示例：

    terrain_randomization:
      height_noise_m: [0.0, 0.01]

2. ground slope

根据 nominal body height 和 leg length 设置：

第一版：

    slope: ±3 deg

不要一开始大坡度。

输出示例：

    ground_slope_randomization:
      roll_deg: [-3, 3]
      pitch_deg: [-3, 3]

3. ground friction

URDF 通常无法可靠给出地面摩擦。
第一版使用：

    friction: [0.6, 1.2]

如果 foot collision 很小、接触面积小，应避免过低摩擦，防止策略过度学滑步。

输出示例：

    contact_randomization:
      ground_friction: [0.6, 1.2]
      restitution: [0.0, 0.1]

4. reset base height noise

根据 nominal_body_height 设置：

    base_height_noise <= 5% nominal_body_height
    但第一版不超过 ±0.02 m

输出示例：

    reset_base_randomization:
      height_m: [-0.015, 0.015]
      roll_deg: [-3, 3]
      pitch_deg: [-3, 3]
      yaw_deg: [-5, 5]

5. push randomization

根据 base mass 设置 push，不要固定乱给。

第一版：

- 小型机器人只给轻微低频 push；
- force 大小按 base_mass * g 的 5%~10% 估计；
- 不要一开始强推。

输出示例：

    push_randomization:
      enable: true
      force_scale_body_weight: [0.03, 0.08]
      interval_s: [3.0, 6.0]

四、基于 URDF 设置 CPG 参数随机性

请用 nominal_leg_length、body_height、joint limit 生成 CPG 参数范围。

1. step_height

根据 nominal_leg_length：

    step_height = 0.08 ~ 0.15 * nominal_leg_length

但第一版限制在：

    0.025 ~ 0.040 m

如果 nominal_leg_length 很小，自动收紧。

1. step_length

根据 nominal_leg_length：

    step_length_max = 0.10 ~ 0.20 * nominal_leg_length

但第一版限制在：

    0.015 ~ 0.060 m

3. frequency

根据目标速度和 step_length 估算：

    freq = cmd_x / step_length

再 clamp 到 motor_profile_fitted 推荐频率。

如果没有真实电机数据，第一版：

    freq_min = 0.8 Hz
    freq_max = 1.8 Hz

4. duty_factor

第一版：

    duty_factor = 0.60

可随机小范围：

    duty_factor: [0.56, 0.66]

但不要太大，避免步态相位混乱。

输出示例：

    cpg_parameter_randomization:
      step_height_scale: [0.85, 1.15]
      step_length_scale: [0.85, 1.15]
      frequency_scale: [0.90, 1.10]
      duty_factor: [0.56, 0.66]
      initial_phase_random: true

五、根据 URDF 自动生成 curriculum

请根据机器人尺寸、质量和电机能力生成分阶段随机性开关。

输出：

    curriculum:
      stage_1_cpg_only:
        randomization_level: none_or_minimal

      stage_2_cpg_residual_basic:
        enable:
          - command_randomization
          - initial_phase_randomization
          - small_reset_noise

      stage_3_motor_random:
        enable:
          - motor_strength_randomization
          - action_delay_randomization
          - pd_gain_randomization
          - joint_zero_offset_randomization

      stage_4_model_random:
        enable:
          - mass_randomization
          - inertia_randomization
          - com_randomization
          - joint_friction_damping_randomization

      stage_5_environment_random:
        enable:
          - ground_friction_randomization
          - small_terrain_noise
          - small_push
          - small_slope

      stage_6_eval_unseen:
        enable:
          - unseen_motor_strength
          - unseen_delay
          - unseen_friction
          - unseen_zero_offset

六、强制安全规则

生成 randomization_profile.yaml 时必须检查：

1. random joint reset 后不能超过 joint limit；
2. zero offset + residual + q_cpg 不能长期贴近 joint limit；
3. step_height 不能导致 calf/thigh 超过安全角度；
4. step_length 不能导致 hip/thigh/calf 目标角接近限位；
5. motor strength randomization 不能让 torque limit 超过 safe torque；
6. mass randomization 不能让 base mass 变化过大导致训练发散；
7. terrain noise 不能超过 step_height 的 25%；
8. randomization 默认从小范围开始，通过 curriculum 逐步放大。

七、接入训练 cfg

请在 fanfan_rl_cpg 的 cfg 中读取：

    config/randomization_profile.yaml

并把其中参数接入：

- event randomization
- reset randomization
- action delay randomization
- motor strength randomization
- Kp/Kd randomization
- mass/inertia/CoM randomization
- terrain/contact randomization
- CPG parameter randomization
- eval unseen randomization

所有参数必须可以在 Python cfg 中覆盖。

八、报告格式

randomization_from_urdf_report.txt 需要写清楚：

1. 哪些随机范围来自 URDF；
2. 哪些随机范围来自 motor_profile；
3. 哪些随机范围来自真机拟合数据；
4. 哪些是默认保守值；
5. 哪些字段缺失；
6. 哪些范围被安全规则收紧；
7. 推荐先开启哪些；
8. 哪些不建议第一阶段开启。

九、第一版建议默认启用

第一版训练默认启用：

- command randomization
- CPG initial phase randomization
- small reset pose noise
- action delay randomization
- motor strength randomization
- Kp/Kd randomization
- joint zero offset randomization

第一版暂时弱启用或关闭：

- large mass randomization
- large inertia randomization
- large CoM randomization
- rough terrain
- strong push
- large slope

目标是：
先提升低速平地小跑对真实电机差异的泛化能力，不要一开始训练复杂越野。
