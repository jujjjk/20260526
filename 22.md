现在需要重新修正 fanfan 四足机器人的 URDF 惯性参数。之前只考虑了补 RS01 电机质量，但现在发现原始 URDF 的质量单位/导出设置存在比例问题：原有结构质量偏小，需要先对原始 URDF 中已有的 link 质量和惯量做统一质量修正。

核心要求：

1. URDF 最终单位仍然是 kg。
2. 原始 URDF 里已有的所有 link mass 需要先乘以 2.20462。
3. 原始 URDF 里已有的 inertia 也要同步乘以 2.20462。
4. inertial origin 不因为这个比例修正而改变。
5. 然后再补 12 个 RS01 电机质量。
6. RS01 电机质量是 0.38 kg/个，不要再乘以 2.20462。
7. 不要修改 joint name、link name、joint axis、joint origin、visual、collision、mesh 路径、limit。
8. 只修改 inertial，包括 mass、origin、inertia。
9. 修改前必须备份原始 URDF。

请按下面步骤做。

第一步：定位 URDF

- 在 urdf/ 目录中找到主要机器人 URDF 文件，优先查找 fanfan.urdf、fanfan*.urdf 或 launch/package 中实际引用的 URDF。
- 如果存在多个 URDF，不要盲改。请优先选择当前用于转换 USD 或训练的那个。
- 修改前备份，例如：
  urdf/fanfan_before_mass_scale_and_motor.urdf

第二步：新建或修改脚本
请新建目录 tools/，如果已存在就复用。

新建脚本：

tools/patch_mass_scale_and_motor_inertia.py

脚本命令形式：

python tools/patch_mass_scale_and_motor_inertia.py \
  --input urdf/fanfan.urdf \
  --output urdf/fanfan_mass_scaled_motor.urdf \
  --base-mass-scale 2.20462 \
  --motor-mass 0.38 \
  --knee-motor-on-thigh \
  --report config/fanfan_mass_scale_motor_report.json

第三步：先做原有结构质量修正
对 URDF 中每个已经有 inertial 的 link 执行：

new_base_mass = old_mass * 2.20462

同时 inertia 也必须同步乘以 2.20462：

new_ixx = old_ixx *2.20462
new_ixy = old_ixy* 2.20462
new_ixz = old_ixz *2.20462
new_iyy = old_iyy* 2.20462
new_iyz = old_iyz *2.20462
new_izz = old_izz* 2.20462

注意：

- inertial origin xyz 不变。
- 这个步骤表示原有 CAD/URDF 结构质量比例修正。
- 这个步骤不要改变 visual/collision 尺寸。
- 这个步骤不要改变 mesh。
- 这个步骤不要改变 joint。

第四步：再补 RS01 电机质量
机器人是 12DoF 四足，每条腿 3 个 RS01 电机。RS01 单个质量为 0.38 kg，12 个合计 4.56 kg。

leg prefix 包括：
FR, FL, RR, RL

每条腿补 3 个电机：

1. hip 电机：
   - 加到 <LEG>_hip link
   - 新增质量 0.38 kg
   - 电机局部位置建议设为该 link 坐标原点附近，例如 [0, 0, 0]

2. thigh 电机：
   - 加到 <LEG>_thigh link
   - 新增质量 0.38 kg
   - 电机局部位置建议设为 thigh link 坐标原点附近，例如 [0, 0, 0]

3. knee/calf 电机：
   - 默认加到 <LEG>_thigh link
   - 新增质量 0.38 kg
   - 原因：不要让 calf 小腿过重，膝关节电机通常更接近大腿/膝部，类似 Unitree A1 的等效建模思路
   - 位置应尽量使用 <LEG>_calf_joint 的 origin xyz，作为 knee motor 在 <LEG>_thigh 坐标系下的位置
   - 如果找不到 calf_joint origin，则使用近似位置 [0, 0, -0.156]

最终每条腿大致是：
<LEG>_hip    = 原结构质量×2.20462 + 0.38
<LEG>_thigh  = 原结构质量×2.20462 + 0.38 + 0.38
<LEG>_calf   = 原结构质量×2.20462
<LEG>_foot   = 原结构质量×2.20462

不要把电机质量加到 foot。
不要把 12 个电机全部加到 trunk。
不要把 RS01 的 0.38 kg 再乘以 2.20462。

第五步：惯量重新计算
不能只改 mass，不改 inertia。

请实现组合刚体惯量计算：

对于每个被补电机的 link：

- 原结构经过质量比例修正后，有：
  m0_scaled
  c0
  I0_scaled
- 新增一个或多个电机：
  mi = 0.38 kg
  ci = 电机在该 link 局部坐标系下的位置
  Ii = 电机自身近似惯量

新质心：
c_new = sum(m_i * c_i) / sum(m_i)

新惯量使用平行轴定理：
I_about_c_new = Σ [ I_i_about_own_com + m_i * ((r·r)E - r r^T) ]

其中：
r = c_i - c_new

电机本体惯量近似：

- 可以把 RS01 近似为半径 0.04 m 的小球：
  I = 2/5 *m* r^2
- m = 0.38 kg
- r = 0.04 m

重点：

- 惯量不能为 0。
- 惯量矩阵要物理合理。
- 惯量需要包含电机相对新质心的平行轴贡献。

第六步：检查脚本
请再新建或更新：

tools/check_urdf_mass_com.py

功能：

- 读取 URDF；
- 输出每个 link 的 mass、inertial origin、inertia；
- 输出总质量；
- 按部分统计质量：
  trunk
  FR leg
  FL leg
  RR leg
  RL leg
- 在零关节角 q=0 下递归计算每个 link 的世界坐标；
- 计算整机 global center of mass；
- 检查左右质量是否对称：
  abs(FR - FL) < 0.02 kg
  abs(RR - RL) < 0.02 kg
- 检查整机 CoM 的 y 坐标：
  abs(com_y) < 0.005 m 优先，放宽不超过 0.01 m
- 检查 inertia 对角线为正；
- 检查惯量三角不等式：
  Ixx + Iyy >= Izz
  Ixx + Izz >= Iyy
  Iyy + Izz >= Ixx
- 如果发现 link 缺少 inertial，只打印 warning，不要自动乱加质量。
